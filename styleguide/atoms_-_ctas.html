Buttons

The following button styles are used for the main call-to-action on the page; any actions which need to stand out and form functionality. By default each type of content area is assigned a style and colour button.

Buttons styles are created in HTML by adding the required button class modifiers to an <a> with a .btn class.

All buttons display to the full width of their parent on mobile (xs).

For small (sm), medium (md) and large (lg) breakpoints buttons display based on the width of ther content.

Each brand colour typically has two additional variations - dark and darker. Buttons use the dark variable background colour for default and the darker variable for the hover state.

See the examples below.

Ghost Buttons

Use ghost buttons when the button is to sit above a coloured background.

All ghost buttons have a 2 pixel white border and white anchor text.

Note: adding a ghost button above a white background will result in an invisible button, so only use them on coloured backgrounds.

A colour class still needs to be added to the button so the anchor text can transition to that colour on hover.

Inline text links

Link usage

Links should be wrapped in 'p' tags for semantic markup. Please use the colour class you wish the link to be as below for the underline colour of the link.

On the wrapping 'p' tag, add the class for the size it should be as the bottom three examples demonstrate. Font weight can also be changed in the same way by using .type--semibold.

There are two main link states below, inline and on their own. The inline links are shown first and the links on their own are displayed after.

Inline Links

For links within paragraphs please use the inline variation of the link. E.g.

Lorem ipsum Inline orange link dolor.

Standalone Links

For standalone links please use the normal link variation. This is primarily for CTA type links:
